# Dog Emotion Detector 🐾

A small web app that guesses whether a dog in a photo looks **happy** or **angry**,
using a MobileNetV2 model fine-tuned on the `Train/angry` and `Train/happy` image sets
from this repo.

## Local development

```bash
cd webapp
pip install -r requirements.txt

# (only needed once, or to retrain) — trains on ../Train and writes backend/model.pt
python3 backend/train.py

uvicorn backend.app:app --reload --port 8000
```

Then open http://localhost:8000

## Project layout

```
webapp/
  backend/
    train.py    # trains the MobileNetV2 classifier, saves backend/model.pt
    app.py      # FastAPI app: /api/predict + serves the frontend
    model.pt    # trained weights (generated by train.py)
  static/
    index.html
    style.css
    script.js
  Dockerfile
  requirements.txt
```

## Deploying to Hugging Face Spaces (free)

1. Create a free account at https://huggingface.co and click **New Space**.
2. Choose **Docker** as the Space SDK, any name (e.g. `dog-emotion-detector`).
3. Push the contents of this `webapp/` folder to the Space's git repo:

   ```bash
   cd webapp
   git init
   git lfs install
   git lfs track "*.pt"
   git add .
   git commit -m "Deploy dog emotion detector"
   git remote add space https://huggingface.co/spaces/<your-username>/<space-name>
   git push space main
   ```

4. The Space will build the Docker image and serve the app automatically at
   `https://<your-username>-<space-name>.hf.space`. Share that link with anyone.

`model.pt` is a few MB — Hugging Face Spaces requires files over 10MB to go through
Git LFS, hence the `git lfs track` step above (safe to run even if the file is smaller).

### Alternative hosts

The same `Dockerfile` also works as-is on Render, Railway, Fly.io, or any host that
runs an arbitrary Docker container and exposes port 7860.
